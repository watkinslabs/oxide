PR-time CI = the only wall. No soak gating, no duration-based exit criterion. AI-driven oracle proptests + miri + loom + QEMU differential are how bugs get found.

## 1 Frozen invariants

1. PR cannot merge without green PR-time CI on both arches.
2. CI calls `xtask` only; no direct cargo invocations in CI scripts.
3. Perf regression >5% on `04§1` hot-path budgets fails CI unless commit has `Performance-Regression-Justification:` trailer.
4. Build env = pinned Docker image (digest, not tag). Image hash in CI logs.
5. Release tag = `00§15` (PR-time green + acceptance binary list pass). No duration-based wait.

## 2 PR-time CI (both arches)

Runs on every PR. THE phase gate.

| Job | Cmd |
|---|---|
| build-kernel-{x86_64,aarch64}-release | `xtask kernel --arch X --profile release` |
| build-kernel-{x86_64,aarch64}-dev | `--profile dev` |
| build-user-{x86_64,aarch64} | `xtask user --arch X` |
| build-image-{x86_64,aarch64} | `xtask image --arch X` |
| qemu-smoke-{x86_64,aarch64} | `xtask qemu --arch X --headless --timeout 60s --expect "init started"` |
| test-hosted | `xtask test --hosted` (incl 10M-op proptests) |
| test-loom | `xtask test --loom` |
| test-miri | `xtask test --miri` |
| qemu-acceptance-{x86_64,aarch64} | runs `43§2` minimum acceptance binaries to clean exit |
| spec-lint | `xtask spec-lint` |
| doc-check | `xtask doc-check` |
| clippy | `cargo clippy --all-targets -- -D warnings` |
| deny | `cargo deny check` |
| bench-vs-history | bench; compare `bench-history/main`; fail >5% regress |
| coverage-gate | per-crate ≥95% on critical (`pmm`,`slab`,`vmm`,`sched`,`vfs`); ≥80% HAL |

Parallel where independent. PR can merge as soon as all jobs are green.

## 3 Release tag CI

`release.yml` triggered on tag push. Verifies:
1. PR-time CI green on the tagged commit, both arches.
2. All `43§2` minimum acceptance scenarios pass on the tag.
3. Kernel-completeness audit `docs/kernel-audit.md` shows no stub regressions vs the sweep landed sessions 38.

Artifact: `(commit, arch, sha256s of kernel + rootfs, acceptance-binary list with exit codes)`.

## 4 Bench history

`bench-history/main/<commit>.json`: per-bench median+p99 cycles. Pruned to last 90d + version tags.

`tools/perfrunner/` cycle-accurate timing (`rdtsc`/`cntvct`), 1000 iters/op, 5 trials.

PR bench job: rerun, compare last `main` commit; fail >5% p99 regress on any `04§1` hot path.

## 5 Docker image (build env)

Single `Dockerfile.build` image. Built once per Dockerfile change (workflow `dockerfile.yml`); pushed to `ghcr.io/<org>/oxide2-build:<dockerfile-sha>`. CI pulls; never rebuilds per PR.

### 5.1 `Dockerfile.build`

Base: `debian:trixie@sha256:<digest>` (digest-pinned).
Layers:
- nightly rustc per `rust-toolchain.toml` + `rust-src` + `rustfmt` + `clippy` + `miri` + `llvm-tools-preview`.
- `qemu-system-{x86_64,aarch64}`.
- EDK2/OVMF firmware (x86_64) + `qemu-efi-aarch64`.
- `mtools`, `sgdisk`, `zstd`, `cpio` (image building).
- `cargo-deny`, `cargo-tarpaulin`, `cargo-fuzz`.
- `git`, `curl`, `make`.

Multi-stage to keep final image lean.

## 7 Runners

- PR-time: GHA hosted (`ubuntu-latest` x86, `ubuntu-24.04-arm` aarch64). Native arm; no qemu-emulation of arm runner.
- Bench-history arm cycles: nightly job on a separate ARM runner (rented hourly when needed) to capture true cycle counts.

## 8 Test contract (frozen)

- PR adding `static mut FOO`: build fails (`07§5`).
- PR regressing `pmm.alloc(0)` p99 ≥10%: CI fails.
- PR adding new spec without `MANIFEST.md` entry: `doc-check` fails.
- Release tag with `43§2` acceptance binary failing on the tagged commit: `release.yml` rejects.
- Docker image build artifact has SHA-256 in CI log every run.

## 9 Failure modes

- GHA hosted runner OOM during build: bump runner size to `ubuntu-latest-8core` (paid). Rare; kernel build fits in 7 GB.
- Bench-history corrupted: rebuild from last good commit's recorded values.

## 10 Debug

CI artifacts (90d retention): full serial log on QEMU jobs, kernel ELF + split debuginfo on build jobs, full bench output on bench job.
