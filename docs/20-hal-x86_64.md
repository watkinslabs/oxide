# 20 HAL x86_64

Status: DRAFT 2026-05-02
Depends on: `01`,`02`,`06`,`07`,`14`,`22`,`23`,`38`.
Provides to: kernel (via `hal` traits).

## 1 Purpose

x86_64 impl of `CpuOps`,`MmuOps`,`Context`,`IrqOps`,`TimerOps`. Boot trampoline. Syscall+IRQ entry asm. Per-CPU plumbing.

## 2 Floor

ISA floor `x86-64-v3` (Haswell+). Mandatory: x2APIC, TSC-deadline, invariant TSC, RDRAND/RDSEED, AVX2 (user only), SMEP, SMAP, NX, PCID, FSGSBASE, XSAVE.

## 3 Boot

Bootloader: Limine (modern protocol). Hands kernel: physical entry point, framebuffer, memory map, ACPI RSDP, SMP info, modules (initramfs).

`_start` (in `boot-x86_64`):
1. Already in long mode (Limine). RIP at `kernel_phys_base`.
2. Set up early GDT (kernel CS/SS, user CS/SS, TSS).
3. Set up early IDT (panic stubs + page-fault stub).
4. Map kernel to higher half (`0xFFFF_FFFF_8000_0000`+); identity map preserved for early access.
5. Switch CR3 to new PML4. Jump to higher-half `start_kernel`.
6. Init PMM from memory map.
7. Init VMM (kernel direct-map at `0xFFFF_8800_0000_0000`+).
8. Bring up BSP per-CPU area (GS_BASE → per-cpu#0).
9. Init APIC, TSC-deadline.
10. ACPI MADT walk → start APs (AP startup at 16-bit real-mode trampoline copied to <1MB; brief switch to long mode then jump to `ap_start`).
11. Each AP brings up its own per-CPU area, joins scheduler idle loop.
12. Continue into kernel `start_kernel` → init drivers, mount initramfs, exec init.

## 4 Memory map

| VA range | Use |
|---|---|
| `0x0000_0000_0000_0000` .. `0x0000_7FFF_FFFF_FFFF` | userspace (47-bit) |
| `0xFFFF_8000_0000_0000` .. `0xFFFF_87FF_FFFF_FFFF` | (reserved) |
| `0xFFFF_8800_0000_0000` .. `0xFFFF_C7FF_FFFF_FFFF` | kernel direct-map (phys to VA: `KDM_BASE + pa`) |
| `0xFFFF_C800_0000_0000` .. `0xFFFF_E7FF_FFFF_FFFF` | vmalloc / kernel modules |
| `0xFFFF_E800_0000_0000` .. `0xFFFF_FE7F_FFFF_FFFF` | per-CPU stacks, IRQ stacks, NMI stacks |
| `0xFFFF_FE80_0000_0000` .. `0xFFFF_FEFF_FFFF_FFFF` | KASLR-randomized kernel image area |
| `0xFFFF_FF00_0000_0000` .. `0xFFFF_FFFF_FFFF_FFFF` | fixmap, vsyscall (vsyscall=disabled), vDSO (mapped into user too) |

## 5 PT format

4-level (PML4 → PDPT → PD → PT). 5-level open question (per `01§OQ`).

PTE bits (frozen):
- `P` (present), `W` (write), `U` (user), `WT`,`CD`, `A`,`D`, `PS` (huge), `G` (global), `XD` (no-execute, bit 63).
- Phys addr in bits 12..51.

`MmuOps::map(va, pa, flags, sz)` chooses PD/PT level based on `sz`:
- 4 KiB → PT entry.
- 2 MiB → PD entry with `PS=1`.
- 1 GiB → PDPT entry with `PS=1`.

PCID:
- ASID 0 = kernel-only PT (KPTI-on path).
- ASID 1..N-1 = user PTs, allocated per-AS.
- `INVPCID` for selective flushes.

## 6 KPTI

Two PML4s per process: `pgd_kernel`,`pgd_user`. Switch on syscall/IRQ entry/exit.
- User PML4: kernel space mapped only for entry trampoline (~16K of code) + per-CPU "trampoline stack" + syscall ABI shim.
- Kernel PML4: full kernel.
- Switch via CR3 write with PCID bit (no flush).

## 7 Syscall entry

`syscall` instruction → `MSR_LSTAR` = `oxide_syscall_entry`.

```asm
oxide_syscall_entry:
  swapgs                            # GS_BASE → kernel per-CPU
  mov  gs:[trampoline_user_sp], rsp # save user RSP
  mov  rsp, gs:[trampoline_k_sp]    # switch to kernel trampoline stack
  push rax                          # save scratch (number)
  mov  rax, cr3
  and  rax, ~0x1000                 # clear KPTI bit → kernel CR3
  mov  cr3, rax
  pop  rax
  # Now on kernel CR3; full kernel mapped. Switch to task's kernel stack.
  mov  rsp, gs:[current_task_kstack]
  push gs:[trampoline_user_sp]      # user RSP
  push r11                          # user RFLAGS
  push rcx                          # user RIP
  push rax                          # syscall nr
  # Build SyscallArgs from rdi,rsi,rdx,r10,r8,r9.
  mov  rdi, rax                     # nr
  call dispatch
  # rax = retval (signed -errno or non-neg)
  pop  rdi                          # discard saved nr
  pop  rcx                          # restore user RIP
  pop  r11                          # restore user RFLAGS
  pop  rdi                          # user RSP
  mov  rsp, rdi
  # Switch back to user CR3 (KPTI).
  mov  rdx, cr3
  or   rdx, 0x1000                  # set KPTI bit → user CR3
  mov  cr3, rdx
  swapgs
  sysretq
```

≤80 lines incl. comments. Cycle-counted; budget per `04§1.1`.

## 8 IRQ entry

Per-vector stub pushes vector and jumps to common entry (per `22§4`). Common entry similar shape to syscall entry but saves full `pt_regs` on IRQ stack. Returns via `iretq`.

## 9 Context switch

`14§5` (already specified). Asm in `crates/hal-x86_64/src/context_switch.S`.

## 10 Per-CPU

`gs:[..]` indexed; per-CPU area allocated at boot for each CPU, GS_BASE set via `WRGSBASE`.

## 11 IrqOps (x2APIC)

```rust
fn enable_line(line:u32);   # APIC LVT or IRQ remap entry
fn disable_line(line:u32);
fn eoi(_:u32);              # always APIC EOI; no per-line distinction
fn set_affinity(line, mask) # IO-APIC redirection entry rewrite
fn alloc_msi(...)           # MSI-X table program; address+data fmt per Intel SDM
fn send_ipi(target, vec)    # write APIC ICR
fn ack(_) -> Some(vector)   # read ISR for spurious detection
```

## 12 TimerOps (TSC-deadline)

```rust
fn monotonic_ns() -> Nanos     # rdtsc + cycles_to_ns
fn set_oneshot(deadline_ns)    # write MSR_IA32_TSC_DEADLINE
fn freq_khz() -> u32           # cached at boot
```

## 13 Concurrency

Per-CPU areas + APIC. SMP startup uses fixed-physical 16-bit real-mode trampoline; serializes on a per-CPU "ready" flag.

## 15 Test contract (frozen)

- Boot test: hello-world kernel boots, prints to UART, `qemu -isa-debug-exit` clean shutdown.
- SMP test: bring up 8 vCPUs, all reach idle loop.
- Syscall round trip: 1M `getpid`; mean cycles within budget; no register clobber (canary scheme `14§8`).
- IRQ flood: 100K `int3` per CPU; no missed EOI.
- KPTI check: kernel addrs not mapped in user PML4 outside the trampoline window.
- vDSO functional: `clock_gettime(MONOTONIC)` from user agrees with kernel `now()` ±100ns.
- ABI review note in `entry.S` cites Intel SDM Vol 3 §5.8 (syscall semantics) and §6.14 (IRQ).
- Coverage of asm: 100% of vector entries hit by integration tests.

## 16 Failure modes

- Triple fault: dump regs on serial, halt.
- AP startup timeout: kassert with stuck-CPU id.
- TSC sync failure (cross-CPU clock skew >1ms at boot): kassert.
