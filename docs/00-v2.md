# 00-v2 Master Plan: oxide2 v2

DRAFT 2026-05-07. Dep:`00`,`02`,`MANIFEST`,`docs/v2/`.

Companion to `00-master-plan.md`. v1 = kernel substrate + minimum userspace. v2 = kernel parity (de-stub the deferred subsystems) + container/observability/hardware-rich substrate. Each v2 phase is independent — no global ordering — except where its spec lists explicit deps.

## 1 Scope

v2 covers everything downstream of `v1.0` — both kernel parity and userspace platform — in one independent-phase ladder:

**Kernel parity (de-stub the audit + v1 non-goals):**
- Kernel surfaces stubbed-graceful in v1 (xattr→ENOTSUP, pkey→0, kcmp→0, etc.) that need real implementations.
- Subsystems v1 categorically excluded (ptrace, IPC, namespaces, io_uring, bpf/seccomp/landlock, modern mount API, fanotify, userfaultfd, perf, coredumps, DRM/KMS+input).
- Network completion (AF_INET6 socket layer + DHCP + DNS + real virtio-net live driver).
- Memory completion (mremap proper, per-PTE mprotect, MAP_SHARED, MADV_DONTNEED).

**Userspace platform (Fedora-class endgame):**
- Real ld-musl (DT_NEEDED, GOT/PLT, RELA/JMPREL, ld.so.cache, dlopen/dlsym).
- Standard libc + NSS + PAM (musl-with-nss, /etc/{passwd,group,shadow}, pam_unix).
- System manager (real PID 1 with service supervision + dependency order).
- Package manager (rpmbuild against our libc, dnf/microdnf install).
- TTY + login flow (agetty per /dev/tty[0-N], terminfo/ncurses, motd/issue).

v2 does NOT cover (those slip to v2.x):
Wayland/GNOME, USB stack, real ACPI runtime + AML interp, NUMA, hibernate/S3, KVM/hypervisor, NFS/CIFS, FUSE, SELinux/AppArmor/IMA full LSM (only hooks land v2 phase 24), DT overlays, full netfilter, vDSO, FSGSBASE, glibc compat surface (set_thread_area i386, IFUNC).

## 2 Entry criterion

v2 phase work begins after `v1.0` ships per `00§15`. v2 phases parallelize freely — they're independent unless their spec lists otherwise — so a v2 phase can land anytime the substrate it touches is ready.

## 4 v2 release ladder

- **v2.0** — phases 18-19, 33-34, 37 land. Network up (AF_INET6 + DHCP + DNS + virtio-net live) + real ld.so + libc/NSS/PAM + agetty/login flow. This is the "Fedora binaries run interactively" milestone.
- **v2.1** — phases 20-23 + 35-36. mremap proper + namespaces + ptrace + io_uring + system manager + package manager. "Container-class workloads + dnf install" milestone.
- **v2.2** — phases 24-29 + 38. bpf/seccomp/landlock + SysV IPC + xattr/ACL + fanotify + userfaultfd + modern mount API + IPv6 net completion. "Hardened systemd / podman" milestone.
- **v2.3** — phases 30-32. perf/ftrace + coredumps + DRM/KMS + input. "Graphical-capable-but-headless-by-default" milestone, gating the v2.x desktop ladder.

Phase ordering inside each release tag: free, gated only by spec-cited deps. Some phases sit in earlier tags than their phase number suggests because they're cheaper or more user-impactful — that's intentional, not a typo.

## 5 v2 release exit criteria

Each v2.N tag ships when:
1. PR-time CI green on the tagged commit, both arches.
2. The phases listed under that tag (§4) have their per-spec §Test contract green.
3. Acceptance binary list extended (`43§3` or successor) — scenarios appropriate to that tag pass on QEMU.
4. Kernel-completeness audit refreshed: no regressions vs prior tag; the phases listed are flipped to real impls.

No duration wait. No soak.

## 6 v2.x

Reserved for desktop / hardware-rich work — Wayland, GNOME, USB, ACPI runtime, NUMA, hibernate, KVM, NFS/CIFS, FUSE, full LSM impls, DT overlays, vDSO, glibc compat surface, full netfilter. Each is its own subsystem; promotion from v2.x to a numbered v2 phase happens when scope is firm.
